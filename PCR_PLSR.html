Principal Components Regression; Partial Least Squares Regression

STA 206 | Fall 2021

November 29, 2021

When there are a large number of highly intercorrelated X variables, one strategy is to produce a (much) smaller number of linear combinations of the original variables that are orthogonal to each other, and then use these new variables in place of the original X variables in the regression. We will discuss two such methods, namely, principal component regression (PCR) and partial least squares regression (PLSR). These two methods differ in how the linear combinations are constructed. They are implemented by the pcr and plsr functions in the pls library.

Like ridge regression, PCR and PLSR are not scale invariant. In practice, the X variables are usually standardized before applying these methods. In the following, we assume the X variables have been standardized and the response Y has been centered.

Principal Component Regression (PCR)

PCR uses the first few principal components of the design matrix (centered and excluding the column of ones) \(\mathbf{X}\) as regressors in the model. In the following we will have a brief review of principal component analysis (PCA). Note that the main application of PCA is for dimension reduction, where the input data are projected to successive directions with maximum variation subject to orthogonality constraints with the previous directions.

Singular value decomposition

Any \(q \times r\) matrix \(\mathbf{X}\) has a so called singular value decomposition (SVD): \[ \mathbf{X}= \mathbf{U} \mathbf{D} \mathbf{V}^T, \] where \(\mathbf{U}\) and \(\mathbf{V}\) are \(q \times q\) and \(r \times r\) orthogonal matrices, respectively, i.e., \(\mathbf{U}^T \mathbf{U}=\mathbf{I}_q, \mathbf{V}^T \mathbf{V} = \mathbf{I}_r\); and \(\mathbf{D}\) is a \(q \times r\) diagonal matrix with non-negative entries \(d_1\geq d_2 \geq \cdots \geq d_{\min(q,r)} \geq 0\) which are called singular values of \(\mathbf{X}\).

Notes:

  • The singular values are uniquely determined by \(\mathbf{X}\). However, \(\mathbf{U}\) and \(\mathbf{V}\) are not necessarily unique.

  • The rank of \(\mathbf{X}\) equals the number of positive singular values, i.e, \(d_1\geq d_2 \geq\cdots \geq d_{rank(\mathbf{X})}>0=d_{rank(\mathbf{X})+1}=\cdots\)

  • \(col(\mathbf{U[,1: rank(\mathbf{X})]})=col(\mathbf{X})\), \(col(\mathbf{V[,1:rank(\mathbf{X})]})=row(\mathbf{X})\), i.e., the first \(rank(\mathbf{X})\) columns of \(\mathbf{U}\) generate the column space of \(\mathbf{X}\) and the first \(rank(\mathbf{X})\) columns of \(\mathbf{V}\) generate the row space of \(\mathbf{X}\).

  • \(\mathbf{X}^T \mathbf{X}=\mathbf{V}\mathbf{D}^T \mathbf{D} \mathbf{V}^T\), so the columns of \(\mathbf{V}\) are eigenvectors of \(\mathbf{X}^T \mathbf{X}\) and the squared singular values \(d_j^2\) (and possibly some zeros) are the corresponding eigenvalues.

Principal components

Now let’s take \(\mathbf{X}\) as the design matrix (centered, excluding the column of ones; \(q=n\): sample size, \(r=p-1\): number of X variables). The \(j\)th column of the matrix \(\mathbf{V}\), \(v_j\), is called the \(j\)th principal component direction/axis of \(\mathbf{X}\), and \[ z_j:=X v_j= d_j u_j, ~~~ j=1,\cdots, d_{rank(\mathbf{X})} \] is called the \(j\)th principal component (PC) of \(\mathbf{X}\).

Notes:

  • The number of PCs equal to \(rank(\mathbf{X})\) equal to the number of positive singular values of \(\mathbf{X}\).
  • The \(j\)th column of the matrix \(\mathbf{U}\), \(u_j \in \mathbb{R}^n\), is the normalized \(j\)th principal component.
  • The \(j\)th column of the matrix \(\mathbf{V}\), \(v_j \in \mathbb{R}^{p-1}\), consists of the linear combination coefficients (a.k.a. loadings) used to construct the \(j\)th PC, \(z_j=\mathbf{X} v_j=d_j u_j \in \mathbb{R}^n\).
  • The squared singular values of \(\mathbf{X}\) (i.e., the eigenvalues of \(\mathbf{X}^T \mathbf{X}\)) are proportional to the (sample) variances of the PCs, \(var(z_j)\). So \(var(z_1) \geq var(z_2) \geq \cdots\).
  • The PCs are orthogonal to each other: \(z_j^T z_l =0\) for \(j \not= l\).

Principal components: interpretation

It can be shown that, the \(j\)th PC direction (loadings) \(v_j\) solves: \[ \max_{v \in \mathbb{R}^{p-1}} var(\mathbf{X}v), ~~~ \text{subject to}~ ||v||_2=1, ~ v^T (\mathbf{X}^T \mathbf{X}) v_l=0, ~~~ l=1,\cdots, j-1. \]

Therefore, the 1st PC \(z_1=\mathbf{X} v_1\) has the largest (sample) variance among all normalized linear combinations of the columns of \(\mathbf{X}\); the 2nd PC \(z_2=\mathbf{X} v_2\) has the largest (sample) variance among all normalized linear combinations of the columns of \(\mathbf{X}\) that are orthogonal to the 1st PC \(z_1\), etc.

PCA illustration: \(n=200, p-1=2\)

PCR vs. ridge regression

It can be shown that the hat matrix in OLS: \[ \mathbf{H}=\mathbf{X}(\mathbf{X}^T \mathbf{X})^{-}\mathbf{X}^ T=\sum_{j=1}^{rank(\mathbf{X})} u_j u_j^T , \] where \(u_j\) is the \(j\)th column of the matrix \(\mathbf{U}\) in SVD of \(\mathbf{X}\). So the OLS fitted values are: \[ \hat{Y}^{ols}=\mathbf{H} Y = \sum_{j=1}^{rank(\mathbf{X})} (u_j^T Y) u_j ~~~~~~~~~~~~~~(1) \] Note that \(\{u_j^T Y: j=1,\cdots, rank(\mathbf{X}) \}\) are coordinates of \(Y\) with respect to the orthonormal basis \(\{u_j: j=1,\cdots, rank(\mathbf{X})\}\) of the column space of \(\mathbf{X}\), \(col(\mathbf{X})\).

Similarly, the smoothing matrix \(\mathbf{S}_\lambda\) in ridge regression: \[ \mathbf{S}_\lambda=\mathbf{X}(\mathbf{X}^T \mathbf{X}+\lambda \mathbf{I})^{-1}\mathbf{X}^ T=\sum_{j=1}^{rank(\mathbf{X})} u_j \frac{d_j^2}{d_j^2+\lambda}u_j^T, ~~~ \lambda > 0 \] So the ridge fitted values are: \[ \hat{Y}^{ridge}_{\lambda}= \mathbf{S}_\lambda Y = \sum_{j=1}^{rank(\mathbf{X})}(\frac{d_j^2}{d_j^2+\lambda} u_j^TY) u_j, ~~~ \lambda > 0~~~~~~(2) \]

Since for \(\lambda>0\), the dampening factor \(0 < \frac{d_j^2}{d_j^2+\lambda}=\frac{1}{1+\frac{\lambda}{d_j^2}}<1\), ridge regression shrinks the coordinates of Y with respect to the orthonormal basis \(\{u_j: j=1,\cdots, rank(\mathbf{X})\}\). Moreover, greater amount of shrinkage is applied to the basis vectors with smaller \(d_j\) (i.e., increasing index \(j\)).

In PCR, the response Y is regressed to the first \(k\) (\(1 \leq k \leq rank(\mathbf{X})\)) PCs, \(z_1,\cdots, z_k\). Since the PCs are orthogonal to each other (and orthogonal to the column of ones due to the centering of \(\mathbf{X}\)), the estimated coefficient of the \(j\)th PC \(z_j=d_j u_j\) is simply \(\hat{\theta}_j=\frac{z_j^T Y}{z_j^T z_j}=\frac{d_j u_j^T Y}{d_j^2}=\frac{u_j^T Y}{d_j}\) and the PCR fitted values are: \[ \hat{Y}^{pcr, k}= \sum_{j=1}^k \hat{\theta}_j z_j =\sum_{j=1}^k (u_j^T Y) u_j,~~~ 1 \leq k \leq rank(\mathbf{X})~~~~(3) \] So PCR simply conducts a hard thresholding (i.e., set to zero) of the coordinates of Y with respect to the basis vectors \(\{u_j: j>k\}\). The number of PCs, \(k\), is a tuning parameter of PCR and needs to be pre-specified or chosen based on some criterion (e.g., through cross-validation). If \(k=rank(\mathbf{X})\), then PCR becomes OLS and no form of shrinkage occurs.

Partial Least Squares Regression (PLSR)

Like PCR, partial least squares regression constructs a set of linear combinations of the original X variables that are orthogonal to each other and uses this set of variables in place of X in the regression. Unlike \(PCR\), in PLSR, the construction process involves the response Y: the linear combination coefficients depend on the strength of the corresponding (adjusted) X variable’s univariate effect on Y.

PLSR algorithm

  • Inputs: centered response vector Y, standardized design matrix \(\mathbf{X}\) , an integer \(1 \leq k \leq rank(\mathbf{X})\)

  • Initialization: set \(\hat{Y}^{(0)}=\mathbf{0}_n\) and \(X_j^{(0)}=X_j, ~~j=1,\cdots, p-1\). Here \(X_j\) is the \(j\)th column of the design matrix \(\mathbf{X}\), i.e., \(X_j\) is the (standardized) observations of the \(j\)th X variable.

  • Subsequent steps: For \(l=1,2,\cdots, k\)

    (a). PLSR component: \(w_l:= \sum_{j=1}^{p-1} ((X_j^{(l-1)})^T Y) X_j^{(l-1)}\).

    (b). fitted coefficient: \(\hat{\theta}_l:=\frac{w_l^T Y}{w_l^T w_l}\).

    (c). fitted values: \(\hat{Y}^{(l)}=\hat{Y}^{(l-1)}+\hat{\theta}_l w_l\).

    (d). Orthogonalization: \[ X_j^{(l)}:=X_j^{(l-1)}-\frac{(X_j^{(l-1)})^T w_l}{w_l^T w_l} w_l, ~~~ j=1,\cdots, p-1. \]

    (Note that, \((X_j^{(l)})^T w_l=0, ~j=1,\cdots, p-1\))

  • Outputs: PLS components \(\{w_l: l=1,\cdots, k\}\); their fitted coefficients \(\{\hat{\theta}_l: l=1,\cdots, k\}\); the sequence of fitted values \(\{\hat{Y}^{(l)}: l=1,\cdots, k\}\).

Notes:

  • \(w_l\)’s are centered, i.e., having zero mean.
  • \(w_l\)’s are orthogonal to each other, i.e., \(w_l^T w_j=0\) for \(l \not= j\). This is due to (a) and (d).
  • \(w_l\)’s are linear combinations of the original X variables with coefficients depending on Y.
  • The PLSR is nonlinear in Y: \(\hat{Y}^{plsr}=S_k(Y)Y\).
  • It can be shown that, the PLSR components have both high variance and high correlation with the response Y.

PCR and PLSR: Illustration

Let’s look at the meatspec data from the faraway library. This data has fat content along with 100 channel spectrum of absorbances measured on \(215\) meat samples. The goal is to build a model using the absorbance measurements (easy to obtain) to predict the fat content (hard to measure through analytical chemistry).

Let’s first load data and relevant libraries:

In the “absorbance vs. index” plot, each curve corresponds to one sample. The absorbance measurement is quite smooth with respect to the (frequency) index , i.e., measurements of nearby frequencies are similar. This leads to highly correlated X variables.

Both are recommended. prcomp returns a list with the singular values (in sdev component), the PC directions/loadings (in rotation component), and the PCs (in x component).

Fit PCR and PLSR

We can fit PCR by the pcr function in the pls library: The arguments center and scale specify whether X and Y should be centered to have zero mean and whether X should be scaled to have unit variance, respectively; The argument ncomp specifies the number of components to be considered: If not supplied, the maximum number of components is used (i.e., \(rank(\mathbf{X})\)). Note that fitted values and fitted coefficients (of X variables) of all successive models will be returned; The argument validation specifies how validation should be conducted (default: “none”): For validation="CV", the default is to conduct 10-fold cross validation with random splits; The segments and segment.type arguments specify CV folds and how segments are created.

For illustration purpose, let’s fit PCR up to the maximum number of components (here the number of X variables: 100):

# Y; conduct 'CV' (5 folds, consecutive segments) fitted values and coefficients
# (of the standardized X) of these k.max models are returned;
meat.pcr <- pcr(fat ~ ., data = train, ncomp = k.max, center = TRUE, scale = TRUE, 
    validation = "CV", segments = 5, segment.type = "consecutive")

Similarly, we can fit PLSR by the plsr function in the pls library:

meat.plsr <- plsr(fat ~ ., data = train, ncomp = k.max, center = TRUE, scale = TRUE, 
    validation = "CV", segments = 5, segment.type = "consecutive")

Choose the number of components by CV plot

We can use cross validation to choose the number of components for the PCR and PLSR models. Let’s look at the CV plot by the validationplot function:

par(mfrow = c(1, 2))
validationplot(meat.pcr, main = "PCR CV plot")
validationplot(meat.plsr, main = "PLSR CV plot")

mfrow = c(1, 1)

These plots suggest that using 15 components is a reasonable choice for PCR and PLSR on this data, whereas using 5 components (suggested by the PCA scree plot) is likely to have worse performance.

PCR and PLSR with 5 components (scree plot suggested number) perform quite decently and similarly as ridge with GCV selected \(\lambda\). With 15 components (CV suggested number), PCR and PLSR perform better. Moreover, unlike OLS and ridge, the RMSPE of PCR and PLSR is not very different on the training set and on the testing set, indicating little overfitting of these two models when the number of components is kept small.
